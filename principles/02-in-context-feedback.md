# In-Context Feedback

- Status: draft
- Domain: Shared
- Claim: A-03
- Family: affordance
- Lens: feedback
- Question: How does the interface confirm that an action was recognized?
- Rule: Confirm each meaningful action in context with immediate feedback.
- Why it works: Feedback reduces uncertainty, duplicate actions, and silent failure.
- Use when: A user submits, copies, saves, deletes, uploads, or waits.
- Avoid when: Feedback is detached from the action or appears too late to be trusted.
- Small example: A save button moves from idle to spinner to success state; destructive actions show undo.
