# Explicit System Models

- Status: draft
- Domain: Shared
- Claim: A-07
- Family: affordance
- Lens: trust
- Question: How does the interface reveal the current state of the system and journey?
- Rule: Surface workflow state and progression continuously so users can predict what comes next.
- Why it works: A visible model lowers cognitive load and increases completion confidence.
- Use when: Multi-step flows, async work, onboarding, account setup, and service journeys.
- Avoid when: State is hidden in menus, timing, or unexplained transitions.
- Small example: Checkout shows the current step, completed steps, next step, and any blocking requirement.
