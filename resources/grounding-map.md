# Grounding Map

## Norman Lens

Treat Don Norman as the base interpretive layer for this repo.

| Concept | Use in this repo |
|---|---|
| Signifiers | Makes actionability visible through cues instead of guesswork. |
| Feedback | Grounds response, confirmation, and state-change checks. |
| Mappings | Tests whether controls map naturally to expected effects. |
| Constraints | Anchors prevention before recovery. |
| Conceptual Models | Supports journey-level understanding of steps and states. |
| Discoverability | Separates visible options from hidden behavior. |
| Error Recovery | Requires errors to be specific, connected, and fixable. |
