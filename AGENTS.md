# AGENTS.md

## Repo

Public notebook for GUI/TUI design principles, guides, resources, and small supporting assets.
Purpose: keep reusable interface thinking from the surrounding workspace without turning this repo into a dump.

## Rules

- Keep entropy low: one file, one idea, one example.
- Keep `README.md` under 150 lines.
- Do not add top-level directories unless the current set fails.
- Prefer markdown first; use SVG or small PNG/WebP only when text is not enough.
- Do not commit bulk inspiration dumps, copied vendor docs, large binaries, raw exports, or duplicate screenshots.
- Prefer links plus a one-line annotation over copied reference material.
- Put durable heuristics in `principles/`.
- Put applied walkthroughs in `guides/`, and link them back to the principle they use.
- Keep `resources/` as short annotated lists or indexes.
- Put assets in `assets/<slug>/` only when they support a note.
- If two notes overlap, merge or delete; do not fork near-duplicates.
- Prefer pruning to archiving.
- If the structure feels heavy, remove files or folders before adding more.

## Default Shape

- State the problem.
- State the rule.
- Show the smallest example.
- Keep prose tight.
