## Sequence

1. Define one real user intent and the shortest successful journey.
2. Walk the flow end to end: entry, browse, filter, compare, detail, handoff, completion.
3. Check every transition for state clarity, trust cues, and recovery paths.
4. Measure where users can stall: no-results states, hidden next steps, and slow transitions.
5. Capture both qualitative failures and quantitative metrics for the same path.

## Exit Criteria

- The next step is always clear.
- The user can recover from failure without restarting.
- Trust signals stay visible at decision points.
- Performance does not break confidence during the flow.
