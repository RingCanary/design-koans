# List And Filter Findability

- Status: draft
- Domain: Shared
- Claims: A-21, A-22
- Type: guide
- Links to principles: [Scannable Content](../principles/07-scannable-content.md), [Findability-First Listings](../principles/10-findability-first-listings.md), [Trust Through Disclosure](../principles/11-trust-through-disclosure.md)

## Checklist

1. Show what the collection contains before asking users to narrow it.
2. Allow multi-select within a filter when compare workflows demand it.
3. Keep filter labels concrete and scannable; avoid category jargon.
4. Expose the card attributes users actually compare.
5. Make empty, narrow, and no-result states recoverable.
6. Preserve trust cues such as freshness, price clarity, and verification during scanning.

## Exit Criteria

- Users can narrow without losing context.
- Users can compare multiple plausible options in one pass.
- No-result states offer a clear recovery path.
